CSS Class & ID

May 5



Intro

There is no right or wrong way about whether to use ID or class on an simple static html pages. You could use both IDs or Classes. As your css gets complicated, you will then have to determine whether to use IDs or Classes.

So, when and why use IDs or Classes ?

IDs

IDs are represented with # (hash) on css. Id's have some limitations:-

  • You can use only 1 id per element
  • One id can only be used once.
<!--You can do this-->
<button id="red">Button</button>

<!--But you cannnot combile 2 ids on one element like this-->
<button id="red large">Button</button>


<!--Using the id again on different element is not a good practice either-->
<div id="red">This is not a good practice</div>

CLASSes

Classes are represented by . (dot) on css files. Unlike IDs, classes can be used as many times as you prefer.

  • Classes can be used while combining with IDs on css or/and html.
  • And two or many classes could be used in one html elements.
<!-- Valid -->
<button class="red">Button</button>

<!-- Valid -->
<button id="red" class="large">Button</button>

<!-- Valid -->
<div class="red">This is totally fine</div>

And on CSS files

/* Its ok to do this */
#red.large{ /* #red is ID and .large is class */
  color:red;
  font-size: 60px;
}

/* Or we may put class infront */
.large#red{ /* #red is ID and .large is class */
  color:red;
  font-size: 60px;
}


/* This is totally ok, both are classes */
.primary.small{
  color : blue;
  font-size: 10px;
}
