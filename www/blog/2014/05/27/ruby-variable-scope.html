Ruby is a Object Oriented Programming Language (OOP) designed by Yukihiro Matsumoto, aka MATZ on/around February 1993.

Why did he design Ruby?

He answers:- source

"I was talking with my colleague about the possibility of an object-oriented scripting language. I knew Perl (Perl4, not Perl5), but I didn't like it really, because it had the smell of a toy language (it still has). The object-oriented language seemed very promising. I knew Python then. But I didn't like it, because I didn't think it was a true object-oriented language â€” OO features appeared to be add-on to the language. As a language maniac and OO fan for 15 years, I really wanted a genuine object-oriented, easy-to-use scripting language. I looked for but couldn't find one. So I decided to make it.

What is Object Oriented Programming Language?

Its a programming language model which is based on the concept of objects, and defines both data type and the functionality of that data type all together. In OOP you can create the relationship between the objects and inherit characteristics from one another.

To learn more about Object Oriented Language Ruby and its nature, we will need to learn more about how to define and use classes, its methods and objects.

Class

In ruby, classes are defined with "class" followed by class name in CamelCase or just the capitalized word if its a single worded class, and the class ends with word end. For Example:-

class HelloWorld
end

#Or
class Hello
end

So we have class, now we need the variables right? Ruby has four different types of variables.

  • Local Variable

    Local variables are defined with just the small letters and underscores. Just like the name suggests, local variables are locally available for use. For example if a local variable is defined within a method, or a loop, it can only be accessed within that method / loop. All the following are local variables:-

    x = 10
amount = 20
height_of_building = 1000
_Age = 25

  • Instance Variable

    Instance Variables are defined with @ sign in front of a letter/word or combination of words separated by underscore. Instance variables are the most commonly used ones. Unlike the class variables, two or more different objects within the same class or the class which inherits from another class are allowed to have different values for the instance variables. For example

    @x = 10
@amount = 20
@height_of_building = 1000

    How it works?

    class MyAge
  @age = 25
  def self.age
    @age
  end
end

class YourAge < MyAge
  @age = 30
end

p MyAge.age == 25 # true
p YourAge.age == 30 # true

  • Class Variable

    Class Variables are defined with double @@ signs followed by letter/word/or combination of word separated with underscore. These variables are not so common, these can be accessed within the class its defined in. And once the value changes, the value of this variable hangs throughout. 

    @@x = 10
@@amount = 20
@@height_of_building = 1000

    How it works?

    class MyAge
  @@age = 25
  def self.age
    @@age
  end
end

class YourAge < MyAge
  @@age = 30
end

p MyAge.age == 25 # false,returns 30
p YourAge.age == 30 # true

  • Global Variable

    Global variables are defined with $ sign followed by letter/word/or combination of word separated with underscore.These variables are accessible to all the methods, classes and objects; regardless of where they are defined.

    $x = 10
$amount = 20
$height_of_building = 1000
