Github cheatsheet - "Git Busy"

Apr 25



" This is not the complete list, but few that I use 90% of the time when working with git/github "

Clonning, Making changes, adding, commiting, and pushing to/from github

~> git clone https://github.com/dahal/demo
~> cd demo
~> touch index.html
~> touch style.css
~> git init
~> git add .
~> git commit -m "Commit message"
~> git push origin master

After each changes:-

~> git status
~> git add .
~> git commit -m "commit message here "
~> git push origin master

After you delete the file

~> git status
~> git add . #(gives you error)
     #Instead:-
~> git add --all
~> git commit -m "deleted file"
~> git push origin master

Adding/commiting/pushing existing repo to github.

~> git init
~> git status (shows you changes)
~> git add --all
~> git commit -m "initial commit"
~> git remote add origin https://github.com/dahal/reponame
~> git push -u origin master #only the first push
 # Rest of the pushes can be done just by:
~> git push origin master

Branching / Merging

~> git branch nameofyourbranch
~> git checkout nameofyourbranch
~> git push origin nameofyourbranch

After making changes

~> git add --all
~> git commit -m "commit message"
~> git push origin nameofyourbranch

To merge

~> git checkout master
~> git merge nameofyourbranch
~> git push origin master
